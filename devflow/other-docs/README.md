# Other Docs (Documentación Externa)

## Propósito

Esta carpeta almacena **documentación de sistemas externos, librerías, APIs y
proveedores** que se integran con el proyecto.

No es documentación generada por el equipo, sino material de referencia de
terceros: manuales de API, especificaciones de protocolos, datasheets, guías
de integración, PDFs de proveedores, etc.

---

## ¿Qué tipo de documentos van aquí?

- Manuales de API de proveedores y servicios externos.
- Especificaciones de formatos de archivo (posición fija, CSV, XML, JSON).
- Documentación de librerías y frameworks de terceros utilizados.
- Guías de integración de servicios externos.
- PDFs y datasheets de proveedores de hardware.
- Términos y condiciones técnicas de servicios contratados.

---

## Organización

Los documentos se organizan en **subcarpetas por proveedor o sistema**:

```
other-docs/
├── proveedor-a/    → Documentación del Proveedor A
├── proveedor-b/    → Documentación del Proveedor B
├── framework-x/    → Documentación de Framework X
└── ...
```

---

## Notas

- Estos archivos son de **solo referencia**. No se modifican.
- Si un documento externo necesita análisis o interpretación, el resultado va
  en `discovery/` como un DISC, no aquí.
- Mantener los archivos en su formato original (PDF, DOCX, etc.) cuando sea posible.
